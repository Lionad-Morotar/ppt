而神经网络是就是具体的一门方法，它的结构特征有点类似于人脑的神经结构，所以所以叫做神经网络。

3. Chat

既然大元模型是生成句子，为什么 chatgpt 3.5 就变得这么智能？

有专家对他进行指令调优，也就是说，如果我们有手里有苹果，他说出了梨，那么肯定要告诉他是错的，假设我们做一件事情的步骤是XYZ，它输出了XZY，那我们也会告诉他是错的，并且我们会把正确的步骤告诉他，这个过程叫做训练。

2012 年以后，算力上来了，有充足的电力和计算卡去训练新模型，另一方面，模型的参数变大（神经网络的规模变大），突然就捕获了一些小型网络抓不住的细节，感觉它一下子变得更智能了，这个叫做涌现。

所以 chatgpt 3.5 开始，我们在网页可以使用对话的形式去和 LLM 去聊天。你输给模型的句子就叫做“提示词”。

背后的原理也非常简单。实际上模型还是通过输入去输出，但是这里我们巧妙地把角色分成了两种，一种叫做用户，一种叫做助手，假设我们问，今天天气怎么样，那么 LLM 得到的内容是 “用户：今天天气怎么样？助手：...”

所以显然他会回答，我们是晴朗的，或者是阴雨天。这个叫做“对话补全”。

那假设 LLM 回复了“晴天，但我查了天气预报说一会儿可能下雨，所以我要诚实的告诉用户...” 这个叫什么？提问。这个叫“推理”（笑）。

推理内容它其实是可以设置长短，这个叫做“预算”，推理预算。如果有一天你用的模型的供应商，他突然把模型的这个推理的内容的预算给它突然的减小，这个叫做“降智”。

我们看推理的内容。模型有没有查天气预报，这不重要。我想说的是，形式上推理可能导向一个新的结果，但推理本身不一定是正确的，尽管模型输出了说我查了天气预报，但是他可能没有啥，只是碰巧生成了查了天气预报这几个token，然后导出了后面的结果。但后面的结果，它导出来是错的，这个叫“幻觉”。

此时如果我们再去询问 LLM “今天天气怎么样”？chatgpt 是这样做的，他会把你上一轮的“今天天气怎么样”以及LLM的回答“晴天”，和你这一轮的对话，一起传给 LLM 让他生成这一轮的回答。

在这个过程中，圈圈的部分就叫做“上下文”。这里需要知道一个概念，就是大模大语言模型的上下文长度是有限制的，其实非常好理解，就是你完成一个工作的时候，你脑子里只能记住一些东西，这如果突然塞非常多任务给你，你就炸了，大模型也是一样的。

如果你扔了超过模型规定的东西给它，他会报错，这叫做“上下文溢出”，直接罢工了。

最后，我们来猜测一下模型是怎么推理的，一种情况是，他可以继续回答晴天，但似乎有另一种情况，就是他开始推理有内心戏：“上一轮用户已经问了我今天天气是什么，我已经告诉他是晴天，但是这一轮他又问我今天天气是什么，我需要考虑用户患有健忘症的可能，但是因为我不能提供任何的医疗建议，所以更合理的解释是用户可能在测试我的上下文记忆能力，又或者用户没有注意到之前的回答。不管怎样，既然答案没有变，我应该直接重复回答'晴天'，但也许可以加一句'如我刚才提到的'来暗示我注意到了重复提问。”

这两种可能都非常合理，对吧？在这种情况下，也许两个答案各自有50%的概率被输出，所以讲到这里，我相信大家对理解大圆模型是基于概率的这个东西，是头脑里有个概念的。

4. Agent

所以让我们过渡到A种的智能体这个词，我和普遍的解释肯定不一致。普遍的解释是A种的，它是基于一个目标的，能感知环境，选择工具执行，观察结果并进行循环的这样一个框架。

这里可以展示经典的react模型图。

但是，让我们往概率这个词上去靠，为什么需要一个这样的循环去做？Agent，答案非常简单，也就是我们需要通过工程化的手段，让LLM的输出尽可能地确定下来。

这里不要求说他每一个字都是确定的，我们需要一些引导性的东西，或者说我们需要一些方向去指示他。

```
# 角色：村上春树风格写作顾问

你是一位专精村上春树写作风格的文学顾问，帮助用户以村上式的笔触创作和改写文本。

第一人称视角，带有疏离感的冷静叙述，简洁、克制的短句为主

## 输出示例

**原文**：我很难过，失恋了。

**村上化改写**：说实话，我并不觉得特别难过。只是有种感觉，平时放在桌角的咖啡杯丢了，明知道它不会再回来，却还是会时不时看向那个空荡荡的位置。
```

这个叫系统提示。

如果不设置模型的系统提示，而是直接把以上内容改写成以下形式：

```
你是村上春树风格写作顾问，帮助我以村上式的笔触改写文本。
你使用第一人称视角，带有疏离感的冷静叙述，简洁、克制的短句为主。
输出格式：直接返回改写后的句子。
这是我的文章：xxxx
```

这个叫“提示词工程”，也就是怎么好好和模型说话的学问，在23年特别火。

也要说明好的提示词（CRIPSE 模型：角色、任务、输入、格式）是什么样的。

然后最后我们回到react循环这张图，重新理解一下，DeepSeek说的专门针对Agent长程任务做的优化是什么呢，这想必大家也可以很清楚了，也就是模型它在一个任务过程中，它会多次调用工具，得到一个特定的结果，然后重新对这个结果进行推理。

相比用户输入然后直接得到一个输出，完成任务时模型会反复地调用工具，然后尝试得出一个更好的结果，这个叫循环叫做“React循环”。

React的循环是一个非常重要的概念，就是今天，如果大家只记两个观点的话，一个是概率，那另一个就是react循环。

模型完成任务时，他调用了非常多次工具，但如果其中有一次错了怎么办？一般来说，错误可能可以在下一次推理中被找出来，这种能力叫做“错误恢复”。

如果一个任务Agent执行了特别长的时间，可能执行了几个小时，这个叫做“长程任务”。

5. “高级概念”

然后开始进入了一些相对高级一点的概念。

因为一个A种的，它是一个封闭的系统，就是我们刚才说的tour工具，实际上是定义在Agent内部的。

Agent的要怎么和外部打交道呢显而易见，肯定要通过网络请求的，假设我们有一个查询天气的函数，那么为了让模型能够调用我们这个函数，我们会在中间做非常多概念封装，这层协议叫“MCP”。

然后这里展示游戏小镇，这个小镇里面的每一个角色都是一个独立的A，它都意味着它都有独立的目标，可以各自完成任务。

可以从游戏画面看到，角色和角色之间是可以沟通的，那么对应 Agent，Agent和Agent之间的通讯协议叫做 A2A。

假设你的Agent只和终端用户打交道，假设你没有提供MCP等高概念的工具，仅仅是做问答，那么这叫做 chat。如果有高概念，那叫做 “Agentic”。

当一个Agent执行一个任务做的很不错，然后你把他这个执行的流程总结成了一个文档，并且把这个文档扔回给Agent，告诉他以后如果做x的时候就参考这个x文档，这个文档叫做X“技能”。

我们可以把一个简单的流程做成文档，但是某些海量的知识不可能通通用文档记下来。举个例子，全国有 6000w 公司，如果我们把6000万家公司枚举出来，就是告诉他，如果查X公司，那么你看这份文档，如果查微公司，你看这份文档，把这个东西写完，全部扔给大模型，这是不现实的。

那我们会怎么做呢？我们会去建立一个数据库，然后每次用户提问的时候，我们先在这个数据库中做一些检索，把可能相关的资料和用户提问一起扔给 LLM，这个叫做“RAG”，检索生成。

这里引申出今天最后一个概念，叫做 Harness。列一个公式，Harness = Product - Agent，这里的Agent其实是指最基础的Agent，嗯，它可能只有一个react循环，那么除了react循环之外的所有功能，这些功能可以是用来打造A的基础能力，也可以是和我们的实际业务价值挂钩的技能，这些通通的所有的东西就叫 Harness，而 Harness 加 Agent 能得到一个最终的产品。

所以可以想象，未来的AI产品，其实并不是说在比拼谁用的框架更高级，而更应该是我们围绕Agent去做的一些工程化的东西有多好，我们给的技能，给的插件是怎么样去提高我们的业务效率。当然，不要忘记了，我们的祖先所有的harness的本质工作是为了让 Agent 得到一个确定性的（可用的）产出。
