## 核心发现：四个维度的一致性结论

### 1. Agent 章节被严重压缩（🔴 全部 4 位审阅员一致指出）

Agent Group 的 9 页幻灯片试图覆盖 Token → Chat → Prompt → ReAct → Tool Call → MCP → A2A → Skills → RAG → Harness，信息过载且大量关键概念被删或压缩为一句话。

**严重缺失的概念**（`design.md` 明确要求但未实现）：

| 缺失内容 | 来源文档 | 影响 |
|---|---|---|
| 涌现（Emergence） | design.md §3.4 | 理解 LLM 能力跃迁的关键 |
| 微调（Fine-tuning） | design.md §3.4 | 理解 ChatGPT 为什么"变聪明" |
| 幻觉（Hallucination） | design.source.md §4.3 | 理解"概率"最生动的实证 |
| 降智 | design.source.md §4.3 | 理解推理预算被削减的后果 |
| 推理预算 | design.source.md §4.3 | 理解模型"内心戏"长短可调 |
| 模型"内心戏"猜测（50% 概率） | design.source.md §4.3 | 最能让人直观理解"概率"的例子 |
| Agentic vs Chat 对比 | design.md §4.4 | 理解 Agent 本质的关键边界定义 |
| CRISPE 完整表格 + 示例 | design.md §4.4 | 听众不知道 CRISPE 每个字母的含义 |
| 村上春树风格写作对比例子 | design.source.md §4.4 | 系统提示 vs 提示词工程最直观的对比 |
| 6000w 公司 RAG 例子 | design.source.md §4.5 | 极具画面感的夸张对比 |
| 游戏小镇 A2A 例子 | design.source.md §4.5 | 生动的画面感描述 |
| OpenClaw 架构图 | design.md §4.6 | 外链不可靠，Harness 概念缺乏视觉支撑 |
| 工具调用时序图 | design.md §4.4 | 工具调用原理缺乏时序可视化 |

### 2. Demo 部分容错率极低（🔴 教学逻辑审阅员标记为最高风险）

30 分钟内完成 Nuxt 建站 + npm 依赖安装 + GitHub Pages 部署，时间容错率极低。网络波动、依赖安装慢、命令报错、GitHub 认证等任一问题都会导致超时。

- 当前 PPT 仅 3 页支撑 30 分钟现场操作
- 虽然有"切到备用成品"的预案，但现场切换会严重削弱"从零到产物"的冲击力
- 建议正式分享前完整跑通 2-3 次并计时[^5]

### 3. 原始讲稿的"温度"流失（🟡 思绪对齐审阅员核心判断）

`slides.md` 从一份"有温度的、带自我调侃的、边想边说的分享"变成了一份"结构严谨但情感温度降低的知识手册"。

**被删除的珍贵瞬间**：

| 原始讲稿 | slides.md 现状 | 风险 |
|---|---|---|
| "这个叫'推理'（笑）" | 删除 | 幽默感流失 |
| "直接罢工了" | 保留，但散落在冷峻布局中 | 口语化程度降低 |
| "这里解释似乎有些牵强" | 删除 | 坦诚度流失 |
| "我还没想好怎么举例子" | 删除 | 过于"完美"，失去真实感 |
| "怎么结束会很有意思呢？要有意思，而价值可以留给其他大佬上" | 正式化为"谢谢"+金句 | 松弛感消失 |
| "小龙虾"等昵称 | 正式化为"网页版 Agent" | 亲切感降低 |

### 4. 5 处 P0 视觉缺失（🔴 视觉呈现审阅员）

| # | 缺失项 | design.md 原文要求 | 现状 |
|---|---|---|---|

| 2 | 工具调用时序图 | "介绍展示AI 操作电脑的时序图" | AgentLoop 组件替代，非时序图 |
| 3 | Why CC 对比截图 | "Claude Code终端界面截图，对比网页版Agent" | 无视觉对比 |
| 4 | 梯度下降示意图 | "这里需要一张梯度下降示意图" | 只有 mermaid 流程图 |
| 5 | 传统 vs 机器学习对比图 | "这里需要一幅对比图" | 文字+代码块，无视觉冲击力 |

## 改进清单（按优先级排序）

### 🔴 P0 — 分享前必须完成

1. **Agent 章节拆分 + 补全概念**
   - 将 MCP/A2A/Skills/RAG 从 1 页拆为 2 页：MCP+Skills 一页，A2A+RAG 一页
   - 新增一页"概率的代价"：涌现 + 微调 + 幻觉 + 推理预算 + 降智
   - 新增一页 Agentic vs Chat 对比
   - 恢复村上春树风格写作对比例子、6000w 公司 RAG 例子、游戏小镇 A2A 例子

2. **补全 OpenClaw 架构图**
   - 将外链图片转为 SVG 组件或内嵌可靠图片

3. **Demo 增加 Plan B 专用页**
   - 超时 5 分钟内 → 跳过部署，展示本地运行效果
   - 超时 10 分钟以上 → 切换到预构建成品，讲解"正常走完会是这样"
   - 让超时切换有 PPT 支撑，听众不会觉得是现场崩了

4. **增加 Group 过渡页**
   - Intro → Why CC："市面上 AI 工具那么多，为什么偏偏是 Claude Code？"
   - Why CC → LLM："选择了一个工具，接下来需要理解这个工具为什么能工作。"
   - LLM → Agent："LLM 的输出是概率性的，工程上怎么让它靠谱地完成复杂任务？"
   - Agent → Demo："理论讲完了，概念对齐了，接下来把它真正做出来。"

5. **LLM 部分扩充 2-3 页**
   - 当前 6 页撑 25 分钟偏薄，增加"涌现能力"页和"幻觉与推理预算"页

### 🟡 P1 — 显著提升质量

6. **梯度下降页升级为可视化示意图**（3D 曲面 + 小球滚动动画）
7. **Why CC 增加终端 vs 浏览器对比图**（mock 截图 + 对比箭头）
8. **End 页 0→1→∞ 做成动画**（Canvas/SVG 数字变形 + 粒子扩散）
9. **补充提示词工程完整示例**（CRISPE 完整表格 + React 性能优化示例）
10. **在 footer-note 中保留口语化讲稿备注**（供现场参考，恢复"可讲性"）

### 🟢 P2 — 锦上添花

11. 高级概念页（MCP/A2A/Skills/RAG）改为 Bento Grid 布局
12. 上下文溢出做成 token 堆积动画
13. 封面"0 和 1 生长"概念视觉化
14. 演示进度条做成动态增长
15. 增加全屏单焦点金句页/过渡页，丰富布局多样性

## 总体结论

`slides.md` 是一个**骨架完整、视觉风格统一**的 PPT，6 个 Group 扉页全部存在，7 个自定义组件实现质量较高。但作为 90 分钟现场分享，存在三个结构性短板：

1. **内容支撑偏薄** — Agent 章节压缩过度，大量 `design.md` 和 `design.source.md` 中明确要求的内容缺失，造成听众理解断层
2. **Demo 容错率过低** — 30 分钟现场构建风险极高，需要更充分的备用方案和排练
3. **"可讲性"不足** — 原始讲稿中最生动的例子和口语化风格流失，需要恢复

**建议行动顺序**：
1. 先补全 Agent 章节的缺失内容（P0）
2. 验证 Demo 全流程并准备 Plan B（P0）
3. 补充关键视觉元素（P0 → P1）
4. 增加 Speaker Notes 恢复口语化风格（P1）
